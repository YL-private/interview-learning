# 浏览器渲染机制

1. 解析HTML和CSS。浏览器首先将接收到的HTML代码解析成DOM树，这是网页的结构表示，包括标签、属性和内容。同时，CSS代码被解析成CSSOM树，代表网页的样式，包括选择器、属性和值。
2. 构建渲染树。DOM树和CSSOM树合并后，浏览器创建渲染树，这是由DOM树和CSSOM树中与页面显示相关的元素组成的。 渲染树的构建过程会忽略掉不需要显示的元素，如隐藏的元素或设置了display属性为none的元素。
3. 布局。布局阶段是指计算元素在页面中的位置和大小。浏览器根据渲染树的结构和样式信息，计算每个元素在页面中的准确位置和尺寸。布局过程是递归进行的，从根节点开始，依次计算每个元素的位置和大小。
4. 绘制。绘制过程是将元素绘制到屏幕上。浏览器根据元素的位置和大小，将它们绘制成像素，然后显示在屏幕上。绘制过程通常通过图形处理单元(GPU)完成，以提高效率和性能。绘制的顺序是按照渲染树的结构进行的，从根节点开始，依次绘制每个元素。
5. 此外，在整个渲染过程中，浏览器的GUI渲染线程与JS引擎线程是互斥的。如果JS执行时间过长，会导致页面渲染不连贯，造成页面渲染加载阻塞。因此，为了提高性能，通常建议将JavaScript代码放在页面底部，并尽量减少JavaScript对DOM构建的影响
