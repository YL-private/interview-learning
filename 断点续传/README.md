## 断点续传前端 Demo（上传续传 + 下载续传）

这是一个纯前端的 Demo（不依赖构建工具），展示两类“续传”：

1. 上传续传：把大文件切成分片，逐片上传，并把“已上传分片索引”保存在 `localStorage`，刷新页面也能继续（上传模式提供 `mock` 和 `server` 两种）。
2. 下载续传：按分片下载，并把已下载分片保存到 `IndexedDB`，刷新页面也能继续；`server` 模式使用 `Range` 请求。

### 使用方式

1. 直接用浏览器打开：`index.html`
2. 上传续传：
   - 上传模式选 `mock`：无需后端，分片会保存到浏览器 `IndexedDB`，演示续传流程即可。
   - 上传模式选 `server`：需要你自己的后端实现（见 `resume-demo.js` 里的注释/接口约定）。
3. 下载续传：
   - 下载模式选 `mock`：演示从浏览器已上传内容“续传下载”。
   - 下载模式选 `server`：需要后端支持 `Range`（HTTP `Content-Range`/`206 Partial Content`）。

### 重要说明

- 这是“前端逻辑演示”。`server` 模式下，最佳实践是后端提供“分片状态查询接口”（例如：`GET /upload/status?fileId=...`），这样才能做到跨设备/跨浏览器的真正续传。
- 当前 Demo 的上传续传状态记录在 `localStorage`，因此“已上传分片”主要用于演示续传流程。
