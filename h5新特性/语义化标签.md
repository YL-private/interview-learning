# HTML5 语义化标签（面试必背版）

核心：用有含义的标签替代无意义的div，使页面结构清晰、可维护性强，适配SEO和无障碍访问，以下为最常考、最常用内容。

一、高频核心语义化标签（必记）

1.header

页面头部、区域头部（如页头、文章标题区）

2.nav

导航栏，包含页面主要导航链接（如主导航、侧边导航）

3.main

页面主体内容，一个页面只能有一个main标签，包含核心内容

4.section

页面区块、章节，用于划分不同主题的内容（如商品列表区、文章章节）

5.article

独立完整的内容（如文章、帖子、评论，可单独脱离页面存在）

6.aside

侧边栏、附属信息，与主体内容相关但非核心（如侧边广告、文章作者信息）

7.footer

页面页脚、区域页脚（如版权信息、联系方式、底部导航）

8.h1~h6

标题层级，h1最高（页面主标题），h6最低，体现内容层级关系

二、其他常用语义化标签（了解即可）

- figure：包裹图片、图表、代码块等独立内容容器

- figcaption：配合figure使用，用于添加图片/图表的说明文字

- mark：标记需要高亮的文本（如搜索结果高亮）

- time：表示时间，可添加datetime属性标注标准时间格式（利于SEO）

- dialog：定义对话框、弹窗（如登录弹窗、提示框）

三、语义化的核心意义

用具有明确含义的标签，替代大量无意义的div嵌套，让HTML结构更具可读性和逻辑性。

四、语义化的优点（面试必问）

1. 可读性强：代码结构清晰，开发者（自己/团队）能快速理解页面布局和内容关系，便于维护和迭代。

2. 利于SEO：搜索引擎能通过语义化标签，快速抓取页面核心内容，提升页面搜索排名。

3. 无障碍友好：屏幕阅读器等辅助工具，能通过语义化标签识别内容结构，帮助残障用户更好地浏览页面。

4. 团队协作高效：统一的语义化规范，减少沟通成本，避免混乱的div命名（如div class="header" 不如直接用header标签）。

五、面试总结（直接背）

HTML5语义化，就是用header、nav、main、section、article、footer等具有明确含义的标签，替代无意义的div，使页面结构清晰；核心优点是提升代码可读性、利于SEO、适配无障碍访问，同时提高团队协作效率。
